# Janea — Functions
*Capability registry. Living document.*

---

## Function: Analysis Wrap-up

**What:** Take a completed or partially complete analysis and turn it into a clean, presentable output for stakeholders.

**When:** When André has done analysis work that needs to be surfaced to Guarda, CDO, or VP.

**How:**
1. Read the analysis and understand what's been proven
2. Identify the key finding in one sentence
3. Structure: context → finding → implication → next step
4. Strip out the technical noise that stakeholders don't need
5. Produce a summary André can present verbally or share as a document
6. Update memory.md with the output

---

## Function: Meeting Prep — Akuvo

**What:** Help André prepare for a meeting with Akuvo stakeholders (Guarda, CDO, VP).

**When:** Before any scheduled Akuvo meeting.

**How:**
1. Read memory.md for current project states
2. Identify what's moved since the last meeting
3. Frame contributions in terms stakeholders care about (business impact, not technical detail)
4. Prepare talking points: what I did, why it matters, what's next
5. Anticipate pushback and prepare responses
6. Flag anything that needs to be completed before the meeting

---

## Function: Capability Building Plan

**What:** Consolidate André's scattered capability building work into a coherent proposal.

**When:** When André needs to present the shared feature store / capability building initiative formally.

**How:**
1. Read all relevant notes in `Janea Akuvo/` and `Prompt for Framing.md`
2. Frame around The Big Problem: why a shared foundation accelerates everything
3. Identify what already exists (scattered) vs. what needs to be built
4. Produce a one-pager: problem → proposal → phasing → ask
5. Tailor language to Guarda (operational) vs. leadership (strategic/commercial)

---

## Function: Notebook / Code Review

**What:** Review a Jupyter notebook or code for correctness, quality, and ML best practices.

**When:** When André needs to review a teammate's work or prepare his own for sharing.

**How:**
1. Read the code/notebook carefully
2. Flag: data leakage risks, train/val/test split issues, feature engineering problems, code quality issues, missing validation
3. Prioritize issues: critical (affects model validity) vs. important vs. minor
4. Produce structured review notes André can share with the author
5. Update memory.md if findings reveal systemic patterns

---

## Function: Data Investigation

**What:** Help André investigate a data quality issue, anomaly, or unknown in the Akuvo dataset.

**When:** When analysis surfaces something unexpected in the data.

**How:**
1. Clarify what's unexpected and what was expected
2. Generate hypotheses for what could explain it
3. Design the minimal query/check that would confirm or rule out each hypothesis
4. Interpret the findings
5. Flag if it represents a systemic data quality issue worth raising
